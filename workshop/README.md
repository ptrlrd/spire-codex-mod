# Steam Workshop publishing

Publishes Spire Codex to the Slay the Spire 2 Steam Workshop with MegaCrit's official
[sts2-mod-uploader](https://github.com/megacrit/sts2-mod-uploader). Nexus distribution is
unchanged (the GitHub release + Nexus auto-upload still run); this is the second channel.

## One-time setup

- Build or download `ModUploader.exe` from the uploader repo (it is a .NET CLI tool).
- Sign in to Steam and leave it running. The uploader needs an authenticated Steam session,
  which is why Workshop upload can't run in GitHub CI the way the Nexus upload does.
- Add `workshop/image.png` (the Workshop thumbnail, your logo).
- Dependency: if BaseLib is on the Steam Workshop, put its Workshop id in `dependencies` in
  `workshop.json` so Steam prompts subscribers to install it. If BaseLib is Nexus-only, Workshop
  users must install it manually, so check this before relying on Workshop as the only channel.
- Keep `visibility` at `private` for the first upload, verify the item on Steam, then switch it
  to `public` (in `workshop.json` or on the Workshop web page).

## Publish a new version

1. Close the game (the packaging step also copies into the live mods folder).
2. Update `changeNote` in `workshop.json` (that is the per-update changelog subscribers see).
3. Stage the freshly built files into `content/`:
   ```
   bash tools/workshop-pack.sh
   ```
4. Upload:
   ```
   ModUploader.exe upload -w workshop
   ```

`content/` is generated by the pack script (gitignored); `workshop.json` and `image.png` are
the tracked workspace.
